# AI Video Studio — Документация проекта

## Описание
AI Video Studio — веб-приложение для генерации видео из текстовых идей.
Frontend-only приложение с моковыми данными и имитацией генерации.

## Технологический стек
- **Next.js 15** (App Router, TypeScript)
- **Tailwind CSS v4** (CSS-first конфигурация)
- **Zustand** (управление состоянием)
- **Three.js** (планируется для 3D-элементов лендинга)

## Структура проекта (FSD)
```
src/
├── app/              # Next.js App Router (страницы, layout)
├── shared/           # Переиспользуемые утилиты, типы, конфиг
│   ├── api/          # Моковые данные
│   ├── config/       # Константы приложения
│   ├── lib/          # Утилиты (cn, formatCredits, mockDelay)
│   ├── types/        # Базовые TypeScript-типы
│   └── ui/           # Базовые UI-компоненты (кнопки, модалки)
├── entities/         # Бизнес-сущности
│   ├── project/      # Проект (сценарий, сториборд, видео)
│   └── user/         # Пользователь
├── features/         # Фичи
│   ├── auth/         # Авторизация (заглушка Google OAuth)
│   └── studio/       # Генерация видео
└── widgets/          # Составные виджеты
    ├── header/       # Хедер с навигацией
    ├── landing-hero/  # Главный экран лендинга
    └── studio-workspace/ # Рабочая область студии
```

## Дизайн
- **Фон**: Синий градиент (тёмно-синий → голубой) в стиле Sui
- **Стиль**: Вдохновлён unseen studio — премиальный, минималистичный
- **Glass-морфизм** для карточек и поверхностей
- **Кастомный курсор** (планируется)

## Ограничения (ЗАГЛУШКИ)
- Генерация видео имитируется через setTimeout
- Авторизация — мок без реального OAuth
- Нет бэкенда, все данные локальные
- Нет реального биллинга/списания кредитов

## Запуск
```bash
cd app
npm run dev
```

## Журнал отладки и изменений (Changelog)
- **Утечки памяти в WebGL (16 Июля):**
  - **Проблема:** Компонент `WireframeGlobe` не удалял созданные `Geometry`, `Material` и `Texture` при размонтировании (напр. при hot-reload), что вызывало быструю утечку видеопамяти.
  - **Решение:** Внедрен массив `disposables`. Теперь в `useEffect` собираются все создаваемые инстансы Three.js объектов, а при unmount вызывается `.dispose()` для каждого элемента. Добавлен `isMountedRef` для предотвращения загрузки текстур в размонтированный компонент.
- **Производительность скролла (16 Июля):**
  - **Проблема:** Резкое падение FPS и медленный скролл из-за чтения `window.scrollY` непосредственно внутри цикла `requestAnimationFrame` (что вызывает синхронный перерасчет стилей браузером - layout thrashing).
  - **Решение:** Чтение `window.scrollY` вынесено в пассивный слушатель события `scroll` с записью значения в `useRef`. Анимация (paralax/rotation) теперь читает значение из `ref` без обращения к DOM, что полностью восстановило 60 FPS.
- **Рефракция (Эффекты Unseen.co) - Итерация 2:**
  - Отказ от применения рефракции только к картинкам, так как это сильно искажало их цвет (RGB-сдвиг).
  - Картинкам возвращен оригинальный `MeshBasicMaterial`, чтобы цвета выглядели естественно.
  - Рефракция перенесена на уровень **всего экрана** через `EffectComposer` и кастомный `ShaderPass`. Теперь весь `canvas` искажается плавной "жидкой линзой", интенсивность которой динамически увеличивается от скорости скролла (как на реальном сайте `unseen.co/world`). Фон оставлен прозрачным.
